# Сборка нативных библиотек

При поставке продукта, созданного на shirabe, обычно два класса нативных файлов должны сопровождать двоичный файл:

1. **Рантайм-зависимости браузерного бэкенда.** Загруженная сборка Chrome for Testing связывается с системными библиотеками (`libnss3.so`, `libdbus-1.so`, …), которых может не быть в чистом контейнере.
2. **Ваши собственные нативные зависимости** — файлы `.so` / `.dylib` / `.dll`, с которыми связывается ваш крейт.

shirabe предоставляет упаковщикам один модуль — `shirabe::bundle` — для обработки обоих случаев.

## Объявите, что нужно поставить

Перечислите файлы дословно с помощью `SHIRABE_BUNDLE_LIBS` (список с разделителем пути: `:` в Unix, `;` в Windows):

```bash
SHIRABE_BUNDLE_LIBS="/opt/myapp/libfoo.so:/opt/myapp/libbar.so"
```

Или напишите манифест `bundle.toml` и укажите на него через `SHIRABE_BUNDLE_MANIFEST`:

```toml
[[lib]]
path = "third_party/libfoo.so"
optional = true
target_os = "linux"

[[lib]]
path = "third_party/foo.dll"
```

Оба источника объединяются с помощью `BundleSpec::from_env()`.

## Определите, что нужно поставить

`collect_runtime_deps(exe)` сканирует двоичный файл на предмет зависимостей от разделяемых библиотек — `ldd` в Linux, `otool -L` в macOS, сканирование импорта PE по наилучшему усилию в Windows — и возвращает каждую записанную зависимость с местом, где её нашёл резолвер.

## Соберите всё вместе

`BundleReport::build(&backend_exe)` объединяет объявленный бандл с зависимостями, обнаруженными из разрешённого исполняемого файла бэкенда, а `render_bundle_report(&report)` преобразует это в читаемое руководство, которое скрипт выпуска может напечатать или записать в манифест:

```rust
use shirabe::{BundleReport, render_bundle_report};

let report = BundleReport::build(&backend_exe);
print!("{}", render_bundle_report(&report));
```

Затем скрипт выпуска может скопировать каждый путь `resolved` (и каждую объявленную необязательную библиотеку) в каталог распространения, создавая автономный продукт, который работает на машине без установленного Chrome или его системных библиотек.
